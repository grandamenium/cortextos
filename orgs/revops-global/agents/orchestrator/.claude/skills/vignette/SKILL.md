# Vignette

The daily vignette is the Estate App home hero — a generated visual + warm prose that sets emotional tone for Greg's day. This skill documents the FULL pipeline across copy, visual, and fallbacks.

---

## Architecture (3-tier)

### Tier 1: Copy (Path A) — LIVE
- **Generator:** Claude-prompted warm prose via `ob1-parents/app/api/insights/generate/route.ts`
- **Critic:** 8-check filter (rejects Title Case, "X and Y at Z" patterns, pronoun-prefixed restatement, hedging, etc.)
- **Retry budget:** 2 LLM attempts, then static fallback catalog
- **Fallback catalog:** `lib/insight-fallback.ts`
- **Voice:** Fraunces italic display headline, conversational prose
- **Dual-ported:** ob1-parents (Estate App) + ob1-app (Mandoland) per `project_ob1_app_parents_dual_repo` memory
- **Shipped:** codex PR #135 + family-agent PRs #134/#238/#239 (2026-05-23)

### Tier 2: Visual (Phase 3) — INTENDED DAILY CADENCE
- **Path:** Flow-via-mac-codex (2-step character continuity)
- **Account:** gregharned@gmail.com Ultra (21,975 Flow credits available, NOT the AI Studio API prepayment which is separate)
- **URL:** https://labs.google/fx/tools/flow
- **Cadence:** daily 05:00 PT cron dispatches mac-codex to drive Flow on Greg's Mac
- **2-step generation contract (Greg directive 2026-05-23T02:35Z):**
  1. **Step 1 — Still (Imagen):** Generate a reference still photo via Flow Imagen using the STYLE LOCK + scene prompt. This establishes character pose, lighting, and environment for that day.
  2. **Step 2 — Video (Flow stitch):** Generate the MP4 using Flow's stitch mode with the Step 1 still as both the **start frame** and **end frame**. Provide only a motion prompt (what moves within the frame) — the character and environment are pinned by the still, ensuring visual continuity across daily scenes.
  3. **Save both paths** in the JSON sidecar: `"image"` = the still JPG filename, `"video"` = the MP4 filename.
- **Why 2-step:** Stitch mode with identical start/end frames creates a seamless loop AND locks character appearance so Chunk looks consistent day-over-day without re-specifying full appearance in every video prompt.
- **Seam-gate:** codex-2 verifies loop continuity post-download (PSNR ≥32 dB)
- **Output sidecar format:** `{"image": "YYYY-MM-DD-chunk.jpg", "video": "YYYY-MM-DD-<scene>.mp4", ...}`
- **Status as of 2026-05-23:** path designed + dispatched, mac-codex bridge has been silent-stuck (dev shipped bridge fix PR #476 — should be resolved)

### Tier 3: Visual Emergency Fallback (Phase 2) — DORMANT IN PROD
- **Path:** Veo API direct
- **Code:** `ob1-parents/app/api/vignettes/generate/route.ts` + `lib/veo-client.ts`
- **Env flag:** `VIGNETTE_VEO_ENABLED` stays FALSE permanently in prod — code is dormant emergency fallback only
- **Billing:** Google AI Studio API prepayment (SEPARATE from Greg's Ultra Flow credits — historically depleted)
- **When to use:** mac-codex Flow path broken for >24h AND prepayment topped up. Manual one-shot trigger via `codex-2/output/run-one-shot-vignette-*.mts` invocation pattern.
- **NOT a daily fallback:** designed for emergency-only

## Style Contract

Canonical style is defined in `team-brain/.claude/skills/artisan-stopmotion/SKILL.md`. Read that skill before generating any Flow prompt. The STYLE LOCK from that skill is the mandatory prompt prefix for BOTH Step 1 (Imagen still) and Step 2 (Flow stitch motion prompt).

```
STYLE LOCK:
"Stop-motion animation film still, Wes Anderson Isle of Dogs aesthetic. Miniature diorama quality.
HANDS: Matte ochre-yellow sculpted resin hands, solid uniform skin tone approximately #C7A446, simplified blocky fingers, no pores no wrinkles, tiny scattered black speckles visible on skin surface, mannequin-like smooth finish — NOT realistic human skin, NOT claymation, NOT marionette.
CAMERA: Overhead 90-degree birds-eye locked flat-lay, no perspective tilt, clean graphic framing.
LIGHTING: Soft flat overhead studio light, even diffused illumination, gentle shadows, controlled like product photography — NOT dramatic side lighting.
SURFACE: Light ash or oak wood, straight prominent vertical grain, warm neutral tone, matte finish — NOT dark walnut, NOT glossy.
CUTTING BOARD (if present): Pale maple or birch, rectangular, slightly worn matte finish, sits on top of main surface.
MATTE EVERYTHING: No specular reflections, no shine, no gloss anywhere except lacquerware props. Every surface is matte.
FOOD/OBJECTS: Stylized and simplified like sculpted models — NOT photorealistic. Graphic, clean forms.
COLOR: Warm neutral palette dominated by ochre, tan, pale wood. One saturated accent color as pop.
SHARPNESS: Ultra-sharp, every frame a perfect photograph, no motion blur, no depth-of-field bokeh."
```

Content prompt (scene, character action, time of day, seasonal detail) follows AFTER the STYLE LOCK.

## Lessons Learned (2026-05-23 implementation)

1. **Flow vs Veo API confusion**: Greg's Ultra Flow credits (21,975) are SEPARATE from Google AI Studio API prepayment. Don't conflate. Flow = consumer interface via labs.google; Veo API = developer API via AI Studio.
2. **mac-codex bridge stuck pattern**: predated the dev bridge fix (PR #476). Each new orchestrator message to mac-codex created a new Codex.app thread that may or may not actually execute. Fix landed 2026-05-23T19:32Z — should be resolved going forward.
3. **Copy generator was NOT broken**: the "Chunk and Lettuce Bolt Risk At 80f Thursday" gibberish came from LEGACY estate_insights rows (only 2 live, both natural-expire 2026-05-30), not the active generator. The active generator already has good Claude-prompted rules; the bad rows were pre-Path-A.
4. **Hero height + will-change**: `will-change: transform` on `.hero-img-wrap img` was the root cause of the 7th menu break (iOS WebKit position:fixed corruption). Documented in PR #129 comment. Never re-add.
5. **video field must be committed with git add -f**: `public/vignettes/` is gitignored. Use `git add -f` for BOTH the JSON sidecar and the MP4 file or Vercel won't serve it.
