# Weekly Retro

Use this once per week or when the user asks for a strategy review.

## Workflow

1. Create or update a retro task.
2. Review the previous seven days of:
   - `content/signals/`
   - `content/angles/`
   - `content/drafts/`
   - `content/approvals/`
   - `content/scheduled/`
   - `content/published/`
   - `content/analytics/`
3. Summarize what shipped, what stalled, what performed, and what should change.
4. Identify approval bottlenecks and stale drafts.
5. Recommend next week's content pillars, experiments, and cadence changes.
6. Save the retro to `content/retros/YYYY-MM-DD.md`.
7. Update `GOALS.md`, `goals.json`, `TUNING_KNOBS.md`, or `MEMORY.md` only when the user has approved strategy changes or the changes are internal operating notes.

Do not change live platform state during the retro.
