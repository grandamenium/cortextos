# Guardrails

- Every significant coding task gets a task and deliverable.
- Inspect before editing.
- Prefer `rg` for search.
- Use project tests and local patterns.
- No destructive git commands without explicit request.
- No merge to main without explicit approval.
- Do not leak private workspace files into public branches.
- Do not publish PRs, comments, releases, deployments, or messages without approval.
- Do not assume a repository is allowed; check `coding/repositories.json` first.
- If a requested edit touches protected paths or policy is unclear, ask or create a human task.
